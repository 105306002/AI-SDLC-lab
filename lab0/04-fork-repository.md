### 知識準備：什麼是 Fork？

**Fork** 是在 GitHub 遠端（雲端）上，將別人的專案完整複製一份到你自己帳號下的操作。

#### 為什麼要 Fork？
* ✅ **自由修改**：你可以隨意修改程式碼、實驗新想法，而不會影響到原始專案。
* ✅ **貢獻回饋**：修復 Bug 或開發新功能後，可以提交 Pull Request 請求合併回原專案。
* ✅ **完整控制**：你對 Fork 出來的專案擁有完整的控制權（包含 Push、Issues 等）。
* ✅ **適合學習**：這是參與開源專案、學習他人程式碼的最佳起點。

#### Fork vs. Clone 的關鍵差異：
| 特性 | **Fork** | **Clone** |
| :--- | :--- | :--- |
| **位置** | 在 **GitHub 遠端伺服器** 之間複製。 | 從遠端伺服器 **下載到本地電腦**。 |
| **權限** | 讓你在雲端擁有一個具備寫入權限的副本。 | 只是在本地建立一份可編輯的程式碼副本。 |

### 實作步驟：使用 Bob 執行遠端 Fork

在這個步驟中，我們不需要手動尋找 URL 或打開瀏覽器點擊按鈕。因為你已經配置了 **GitHub MCP**，Bob 可以直接調用 API 在你的帳號下建立副本。

#### 1. 啟動 Bob Shell
開啟終端機，輸入 `bob` 進入互動模式。

#### 2. 執行自然語言 Fork 指令
請切換至進階模式，並直接對 Bob 說出你的意圖。請輸入以下指令：

```bash
/mode advanced
使用 GitHub MCP 幫我把這個專案 Fork 到我的帳號：
(https://github.com/105306002/modernization-demo)
```

#### 3. 觀察 AI 動作
Bob 會自動執行以下步驟：
- 🔍 解析專案 URL
- 🔗 透過 GitHub MCP 調用 Fork API
- ✅ 在你的帳號下建立專案副本

#### 4. 驗證結果
完成後，你可以：
- 前往你的 GitHub 帳號查看 Fork 的專案
- 或使用 Bob 查詢：`列出我的所有 repositories`
